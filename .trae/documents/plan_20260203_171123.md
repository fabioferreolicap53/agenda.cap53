# Sincronização em Tempo Real do Status de Participação

## 1. Frontend: Re-fetch do Evento no Modal (`Calendar.tsx`)
- No componente `EventDetailsModal`, adicionar um novo estado `currentEvent` para armazenar a versão mais recente do evento.
- No `useEffect` de inicialização do modal:
  - Realizar um `pb.collection('agenda_cap53_eventos').getOne(event.id, { expand: 'user,location,participants' })`.
  - Atualizar `currentEvent` e resetar o `participantStatus` com os dados recém-buscados.
- Substituir o uso do prop `event` por `currentEvent` em todo o modal para garantir que a interface reflita os dados mais novos.

## 2. Frontend: Garantia de Sincronização Local
- Certificar-se de que a função `fetchEvents` do componente pai (`Calendar`) também seja chamada após certas ações (como excluir ou cancelar) para manter a grade do calendário atualizada.
